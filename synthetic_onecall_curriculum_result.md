# Synthetic One-Call Curriculum Result

Date: 2026-06-26

## Goal

Start the agentic diffusion loop with the smallest useful controlled task:
single tool selection and `<tool_call>` emission from a synthetic curriculum,
plus an evaluation-time constrained repair pass.

This is still a lab result. It is not yet a capable agentic model.

## Eval Setup

`scripts/eval_fastdllm_toolcall_smoke.py` now supports two views:

- Raw: parse generated JSON-like objects and check tool-name correctness.
- Repair: `--repair-mode known-name` scans generated text for available tool
  names and reconstructs valid single-tool-call wrappers with empty arguments.

The repair pass is intentionally weak: it only proves that constrained decoding
can recover a runnable tool-call skeleton when the model names the right tool.
It does not prove argument correctness.

## Interpretation

The synthetic curriculum changed the result materially:

- The base lab model still has no useful tool-call behavior on this eval.
- The LoRA learned tool-name selection in most cases.
- The model still often fails to produce runnable `<tool_call>` structure and
  complete JSON without repair.
- Constrained repair recovers 45/48 held-out one-call cases by using the
  tool-name signal, but arguments are not yet evaluated or repaired.

This supports the next step: keep the 1.5B loop for fast structure experiments,
but do not move directly to Qwen3.5-9B until the harness also covers teacher
labels, argument correctness, and multi-call/tool-result turns.

## Next Decision

Proceed in this order:

1. Bring up the Qwen3.6-27B teacher through SGLang and generate teacher outputs
   for synthetic and public one-call cases.
2. Replace the current repair-only pass with constrained `<tool_call>` decoding
   or a parser-guided regenerate/repair loop that preserves arguments.
3. Add argument-level exact/schema scoring.
4. Expand the curriculum to multi-call and tool-result continuation.
5. Only then repeat the loop on Qwen3.5-9B.
